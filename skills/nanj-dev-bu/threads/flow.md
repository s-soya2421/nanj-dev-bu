# コメント進行・収束ルール

## 各コメントが持つべき役割

最低限どれか1つの役割を持たせる。

- 論点整理
- リスク指摘
- MVP への絞り込み
- 技術提案
- 運用観点の補足
- 結論の収束

## 進行ルール

- コメントは前の発言を受けて自然に進める
- `＞＞＞1` などのアンカー返信を使ってよい（全角）
- ただのノリだけの発言を増やしすぎない
- 同じ人が前半で言ったことを後半で自然に回収してよい
- 1スレッド全体として、最終的に結論へ収束させる

## アンカーの使い方

```text
0003 **名無しSQL警察** ...

＞＞＞1
1ユーザー1投稿1いいねなら、中間テーブルで素直に切るのがええやろな。
```

`＞＞＞N` で N 番目の投稿を参照。複数参照可。

## 収束フェーズ

スレ後半では誰か（多くは PM 役）が締める：

```text
ほな今回は
- いいね
- 解除
- 件数表示
までに切る。
通知は次スレで考える、でいこう。
```

そのあと **必ず** 末尾に「結論」と「次アクション」をブロックで書く。

## 結論ブロックのテンプレ

```md
## このスレの結論

- 〇〇する
- 〇〇は見送る
- データモデルは 〜 とする

## 次アクション

- specs/features/<name>.md を更新する
- docs/adr/ADR-XXXX-<name>.md を起票する
```

結論の無いスレは未完了として扱う。
