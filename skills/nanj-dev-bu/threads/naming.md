# ファイル名・タイトル規則

## ファイル名

形式は真面目につける。

```text
YYYY-MM-DD-<topic>.md
```

例:

```text
2026-04-09-like-feature.md
2026-04-09-auth-flow.md
2026-04-10-feed-ranking.md
```

## スレッドタイトル

タイトルは少し遊んでよい。プレフィックスでカテゴリを示す。

例:

- `# [feature] いいね機能作りたいンゴ`
- `# [adr] 認証方式どうするんや会議`
- `# [feature] Feed ranking のMVP切るスレ`
- `# [refactor] あの神クラスどう割るんや`

## プレフィックス

- `[feature]` — 機能検討
- `[adr]` — 設計判断の論点整理
- `[refactor]` — リファクタ方針
- `[infra]` — インフラ・運用方針
- `[bug]` — 障害分析

## スレを作るべきケース

- 新機能のMVPを切るとき
- 技術選定の比較をするとき
- 実装方式に複数案があるとき
- データモデルや責務分離で迷いがあるとき
- ADR 化する前に論点整理したいとき

## スレを作らなくてよいケース

- 軽微な typo 修正
- 明らかなリファクタ
- 議論の余地がない単純作業
- ノイズしか残らない雑談
