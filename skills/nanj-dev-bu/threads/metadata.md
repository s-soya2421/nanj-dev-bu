# 投稿メタデータ規則

## 投稿番号

- 4 桁ゼロ埋めの連番
- 例: `0001`, `0002`, `0003`, ..., `0099`, `0100`

## 日時

- デフォルトはプロジェクトのローカルタイムゾーンを使う
- 形式: `YYYY/MM/DD(曜) HH:MM:SS <TZ>`
- 例: `2026/04/09(木) 12:03:17 JST`
- タイムゾーンが混在するチームでは UTC 併記を推奨
  - 例: `2026/04/09(木) 12:03:17 JST (UTC: 2026-04-09T03:03:17Z)`

## ハンドル名

- スレッド内でぶらさない
- 同じハンドルは同じ立場・口調・問題意識を維持する

## 投稿行フォーマット

```text
NNNN **ハンドル名** YYYY/MM/DD(曜) HH:MM:SS <TZ>

（本文）

---

NNNN **ハンドル名** YYYY/MM/DD(曜) HH:MM:SS <TZ>

（本文）
```

- ハンドル名は太字（`**...**`）で囲む
- ヘッダ行と本文の間に空行を入れる（Markdown で改行が潰れないようにする）
- コメント間は `---`（水平線）で区切り、ひとまとまりを明確にする

例:

```text
0001 **風吹けば名無しPM** 2026/04/09(木) 12:03:17 JST

いいね機能作りたいンゴ。
ただ通知まで入れると広がるから、まずは投稿への単純ないいねだけで切りたい。
```

## 時系列

- 同一スレッド内では時系列が自然につながるようにする
- 数分〜数十分単位で進めるのが自然
- 1つのスレが何時間にもまたがる場合、間が空くのは構わない
