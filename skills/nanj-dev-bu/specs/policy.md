# Spec Policy

`specs/` は現在の仕様の正本である。

## 書き方ルール

- **掲示板風にしない**
- 実装者が読んで迷わないことを優先する
- 受け入れ条件・制約・挙動を明示する
- 途中の迷いや没案は書かない
- 必要なら ADR と thread を関連リンクとして参照する

## 推奨セクション

```md
# <機能名>

## 概要
何をする機能か、1〜2 段落で。

## 受け入れ条件
- [ ] 条件1
- [ ] 条件2

## 挙動
正常系・異常系の挙動を明示する。

## 制約
- データ的制約
- パフォーマンス制約
- 外部依存

## 関連
- docs/adr/ADR-XXXX-<topic>.md
- history/threads/YYYY-MM-DD-<topic>.md
```

## やってはいけないこと

- 没案や検討過程を書く（→ thread に書く）
- 「なぜそうしたか」の長文説明（→ ADR に書く）
- 曖昧な表現（「だいたい」「適宜」など）
- TODO コメントを正本に残す
