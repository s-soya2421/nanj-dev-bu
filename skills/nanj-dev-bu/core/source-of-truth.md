# Source of Truth

文書の優先順位は以下。

1. **`specs/`** — 現在の仕様の正本。実装者が最終的に従うべき内容を書く
2. **`docs/adr/`** — 設計判断・採用理由・影響範囲の正本。「なぜそう決めたか」を簡潔に残す
3. **`history/threads/`** — 掲示板風の検討ログ。議論・比較・迷い・途中経過を残す。**正本ではない**

掲示板風ログはあくまで**検討過程の保存先**であり、仕様の正本にしてはならない。

## パス運用

- `specs/features/` / `docs/adr/` / `history/threads/` はデフォルト値
- 既存リポジトリで別構成を使う場合は、パスマッピングを明文化してその設定を優先する

## ディレクトリの役割

```text
specs/
  features/           # 機能仕様・受け入れ条件・制約

docs/
  adr/                # アーキテクチャ判断・技術選定の理由

history/
  threads/            # 掲示板風の議論ログ
```

- `specs/features/` — 現在の期待挙動、実装前提、制約、受け入れ条件
- `docs/adr/` — アーキテクチャ・データモデル・外部サービス採用・技術選定の理由
- `history/threads/` — 機能検討、技術方式比較、MVP切り分け、懸念整理、設計会話
